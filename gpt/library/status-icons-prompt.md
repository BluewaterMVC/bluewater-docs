## ✅ Status Icon System

Each documentation file must include a `🧮 Status:` field in its metadata header. This field uses one of the **approved status labels below**, always paired with its corresponding emoji icon.

---

### 🗂 Status Definitions Table

| Icon | Label              | Description                                                     |
|------|--------------------|-----------------------------------------------------------------|
| 🚧   | Stub / Placeholder | Initial stub file with minimal or no content                    |
| 📝   | In Review          | Under active review, pending approval or edits                  |
| ⏳    | Planned            | Scheduled for future development; not yet drafted               |
| 🔐   | Internal Only      | For internal team documentation not meant for public release    |
| ✍️   | Draft              | Actively being written; content is forming but not final        |
| ✅    | Approved           | Reviewed and accepted; may be published or used as a reference  |
| 📘   | Final              | Complete and published; no further changes expected             |
| ⚠️   | Deprecated         | Replaced or phased out; maintained for legacy reference         |
| ♻️   | Needs Update       | Outdated or misaligned with latest standards; requires revision |
| 🗑️  | Obsolete           | No longer relevant; should be removed or archived               |

---

### 📌 Implementation Notes

- **Only one status** may be used per document header.
- Always use the full label and emoji (e.g., `✍️ Draft`).
- Status appears in the metadata block under `🧮 Status:`
